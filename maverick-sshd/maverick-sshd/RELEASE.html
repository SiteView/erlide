Release Notes

The SSHD Maverick Component is designed specifically for intensive, server-side SSH applications. Built to take advantage of the optimized classes provided by the Java Development Kit version 1.4.2. The new Java NIO API provides a range of enhancements that bring maximum performance to your platform-independent security solutions.

Features

The current release provides the following features:

    o  Supports SSH2 protocol
    o  Scalable implementation based upon the Java NIO framework

    o  Optional use of JCE
    o  Configurable number of data transfer, accept and connect threads
    o  Local/Remote port forwarding
    o  Password authentication   
    o  Public key authentication
    o  Blowfish/3DES Encryption
    o  Optional Twofish, AES & CAST Encryption
    o  SHA1/MD5 Message Authentication
    o  ZLIB Compression
    o  NativeAuthenticationProvider interface allows customization of the login process
    o  Abstract SessionChannel allows customization of process execution
    o  Supports SFTP and SCP  
    o  AccessManager interface allows customization of access rights on a user-by-user basis
    o  Optional NativeFileSystemProvider interface to customize file access
    o  Built in VirtualFileSystem provides platform independent NativeFileSystemProvider
       implementation
    o  VirtualFileSystem allows mounts to be defined by mapping virtual directories to
       physical folders

System Requirements

The Maverick SSHD API requires the latest version of the J2SE 1.4.2_05 SDK from Sun Microsystems.

Known Issues

°    Using Maverick SSHD with JDK versions lower that 1.4.2_05

Maverick SSHD uses the Java NIO framework and several bug fixes were provided by Sun since the original 1.4.2 releases that provide a much more stable framework. We recommend that users ONLY use JDK 1.4.2_05 as this elimates many of the problems that were experienced by users using lower revisions of the JDK.
 

Frequently Asked Questions

      °    Where do I start?

Take a look at the API specifications, most notably the Daemon class as this provides base for your SSH server. There are also an example server provided in the examples subdirectory of the distribution file.
