Important Notice

This version of the SSHD Maverick API requires activation through the installation of a license key. You can obtain a license by registering at http://www.sshtools.com

 

Once you have obtained your license key, you should paste the license key code provided into your implementation before any calls to the SshDaemon.

Introduction

Thank you for downloading this release of the SSHD Maverick API.  SSHD Maverick is a development library for creating scalable SSH2 server applications.

System Requirements

The SSHD Maverick API uses the Java NIO framework and requires Sun’s Java SDK 1.4.2_05 or greater. Although earlier versions of the 1.4 SDK support the NIO framework, you may experience problems using our API with them due to problems within the SDK. We recommend and will only support usage of the Java SDK 1.4.2_05 or greater.

Documentation

The SSHD Maverick Documentation contains the API specifications.

Contents

This section contains a general summary of the files and directories in the Maverick SSHD API. 

/examples       A simple server that demonstrates the basics of setting up your own SSHD.

/dist                 JAR files containing the API for use with the Java 2 Standard Edition 1.4.2

maverick-sshd.jar – Contains the server implementation without debug information.

maverick-sshd-debug.jar – This is the debug version of the API and requires that the commons-logging.jar library is present in the class path.

/docs                Contains the API specifications.

/lib                   Maverick SSHD dependencies.

/src                  Java programming language source files for the SSHD implementation; this is only available in the source code version of the product.

/build               An ant script for building the source code. This is only available in the source code version of the product.

Getting Started

Implementing a server using the SSHD Maverick API is simple and quick. We have provided an example that demonstrates the minimum requirements of creating a scalable SSH server. In the /examples folder is an ant script, executing the script will compile the classes and then attempt to run the server on both the standard port of 22 and an alternative port 10022. Make sure you have installed your license key in the ExampleSSHD.java file (look for the comments instructing you where to place it). You can change the port by amending the following lines in the ExampleSSHD.java configure method:

 

context.addListeningInterface("0.0.0.0", 22);

context.addListeningInterface("0.0.0.0", 10022);

We recommend that you examine the ExampleSSHD.java and then take a look at the Daemon and SshContext documentation to get you started in developing your own server implementation.
